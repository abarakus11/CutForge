# Instruções para agentes

## Commits

**Toda alteração no código deve gerar um commit git** antes de encerrar a tarefa.

- Rodar `git status` e `git diff` ao finalizar
- Commitar apenas arquivos relevantes (nunca `.env.local` nem segredos)
- Mensagem clara: `feat:`, `fix:`, `chore:`, etc.
- Push só quando o usuário pedir explicitamente
